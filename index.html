HTML

HTML står för HyperText Markup Language. HTML är som strukturen för en webb sida som skelettet. Man startar alltid ett HTML-dokument med att skriva <!DOCTYPE html>. Det är en instruktion till webbläsaren som talar om att det är en modern webbsida som ska visas.

En annan viktig del är taggen <html lang="sv">. Den talar om vilket språk webbsidan är skriven på. Om man vill ha en engelsk sida ändrar man "sv" till "en", och det finns liknande koder för alla andra språk.

För att få fram text på sidan använder man olika taggar. Två av de vanligaste är: <h1>: Används för att skapa en huvudrubrik. Den kan placeras högst upp eller där man behöver en tydlig rubrik. <p>: Används för vanlig text (paragrafer).

När man skriver HTML använder man alltid tecknen (<) och (>) (vinkelparenteser) för att skapa taggar. De flesta taggar behöver också en sluttagg, till exempel </h1>, för att berätta för webbläsaren var rubriken slutar.

För att verkligen förstå hur en webbsida är uppbyggd behöver man ha koll på några specifika taggar här är några andra exempe och hur de fungerar:

<head>: Det här är sidans "hjärna". Allt som skrivs här inne syns inte för besökaren (förutom titeln i fliken), men det är här vi laddar in typsnitt, länkar till CSS och berättar för Google vad sidan handlar om.

<meta charset="UTF-8">: Detta är en av de viktigaste raderna i head. Den talar om för webbläsaren att vi använder teckenstandarden UTF-8, vilket gör att svenska tecken som å, ä och ö visas på rätt sätt och inte som konstiga symboler.

<style>: Om man inte vill använda en separat fil kan man skriva sin CSS direkt här inne. Det är smidigt för små sidor, men blir rörigt på stora projekt som den som jag jobbar med nu. Oftast så brukar man att göra css/style i en separat fil.

<body>: Detta är själva kroppen på sidan. Allt du ser från text, bilder, knappar och videor/länkar måste ligga inuti denna tagg för att visas i hemsidan.

<div>: En "division" eller box. Det är en osynlig behållare som vi använder för att gruppera ihop saker så att vi kan flytta runt dem eller färglägga dem med CSS. Div har jag använr för att organisera min hemsida och dela upp den på att bar sätt.

<span>: Liknar en div, men används oftast inuti en textrad om man vill styla ett enstaka ord, till exempel göra just det ordet rött eller fetstilt.

CSS

CSS står för CascadingStyle Sheets. Om HTML är skelettet/strukturen som håller upp innehållet, så är CSS webbens kläder och design för att göar webben finare. Det är det som gör att hemsidan ser proffsig ut. Med CSS kan man bestämma allt ifrån färger och typsnitt till hur stora bilder ska vara och var på sidan de ska ligga.

Man börjar med att skriva "regler" för hur olika delar av HTML-koden ska se ut. En regel består av en selector (vad du vill ändra) och en declaration (vad ändringen ska vara).

Till exempel, om du vill att alla dina rubriker (<h1>) ska vara blå, skriver man: h1 { color: blue; }

Det finns tre sätt, men det absolut vanligaste och bästa sättet är att länka en separat CSS-fil till din HTML-fil.

Skapa en fil som heter style.css.

I din HTML-fil, inuti <head>-taggen, lägger du till denna rad: <link rel="stylesheet" href="style.css">

Att separera HTML och CSS är en av de viktigaste grundpelarna inom modern webbutveckling

Här är varför vi håller isär dem och vilka fördelar det ger.

1. Genom att dela upp koden får varje del ett tydligt syfte:

HTML (Struktur): Definierar vad innehållet är (rubriker, stycken, listor).

CSS (Presentation): Definierar hur innehållet ska se ut (färger, typsnitt, placering).

2. Enklare underhåll: Om du vill ändra utseendet på hela din webbplats behöver du bara redigera en enda CSS-fil istället för att gå in i varje enskild HTML-sida. Detta sparar tid och minskar risken för fel.

3. Snabbare laddningstider: Webbläsaren sparar CSS-filen lokalt (cache). När en besökare går mellan olika sidor på din webbplats behöver bara det nya innehållet laddas, vilket gör att sidan upplevs som snabbare.

4. Renare kod: Genom att flytta bort all designkod från HTML-filerna blir de mycket mer lättlästa och strukturerade, vilket gör det enklare för både dig och andra utvecklare att samarbeta i projektet.

JavaScript

Vi vet att HTML är skelettet för en hemsida och att CSS är webbens kläder/design då är Javascript som hjärnan och musklerna, de lägger till extra funktioner som dark mode och light mode.

Medan HTML definierar innehållet och CSS stilen, gör JavaScript webbsidan interaktiv. Det är ett programmeringsspråk som körs direkt i webbläsaren hos den som besöker sidan.

JavaScript fungerar genom att webbläsaren läser din kod och omvandlar den till instruktioner som datorn kan utföra direkt på din skärm. Man brukar säga att JavaScript är ett tolkat språk, vilket innebär att man kan fatta språket enkelt.

Java och JavaScript är två populära programmeringsspråk som ofta blandas ihop eftersom deras namn liknar varandra. Trots det fungerar de på olika sätt och används till olika saker inom programmering.

Java är ett kraftfullt programmeringsspråk som används för att skapa stora program, appar och system. Det används ofta inom företag, banker och Android-appar. Java är känt för att vara stabilt och säkert, vilket gör att många stora företag använder det i sina system. När man programmerar i Java behöver man skriva ganska tydlig och strukturerad kod.

JavaScript används främst på webbsidor för att göra dem interaktiva och levande. Med JavaScript kan man till exempel skapa knappar som reagerar när man klickar på dem, animationer och spel i webbläsaren. Nästan alla moderna hemsidor använder JavaScript på något sätt. Språket är enklare att börja med för många nybörjare eftersom man snabbt kan se resultat direkt i en webbläsare.
